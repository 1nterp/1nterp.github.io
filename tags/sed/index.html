    bash 에서 따옴표를 쓸 때

    bash 쉘 스크립트를 작성할 때, 현재 디렉토리에 있는 파일 중 $1 문자열을 $2 문자열로 치환하라는 명령을 작성했다. 그런데 $1과 $2가 인식이 안 된다. 정확히 말하면 쉘이 ‘$1’ 이라는 문자열을 찾아 ‘$2’ 라는 문자열로 바꾸려고 시도했단 것이다. 이유가 뭘까? 바로 sed 명령을 작은 따옴표로 사용했기 때문이다. 우선 방금 이야기한 간단한 스크립트를 보자. 작은 따옴표 안에서 변수를 참조하고 있는데, 잘 안 된 것이다. Bash Reference 의 작은 따옴표/큰 따옴표 설명을 보면, 다음의 규칙이 존재한다.
